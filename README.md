# 未入金・支払い催促実務ガイド

取引先からの未入金・支払い遅延に悩む中小企業の経理担当者、フリーランス、制作会社、受託開発者のための実務ガイドです。

## サイトの目的

このサイトは、「請求書を送ったのに入金がない」「催促メールをどう書けばいいかわからない」という悩みを持つ方に向けて、実務で使える催促メールのテンプレートと対応フローを提供します。

### 想定読者

- 取引先から入金されていない中小企業の経理担当者
- フリーランスとして活動しており、クライアントからの支払いが遅延している方
- 制作会社・受託開発会社で、未払い請求を抱えている方
- 「もう2回催促したのにまだ払ってくれない」とリアルに困っている方

### 読者の悩みフェーズ

本サイトでは、読者の状況を2つのフェーズに分けてサポートします：

**フェーズ1: まだ角を立てずに丁寧に催促したい**
- 1〜2回目の催促段階
- 相手との関係を維持しながら、やわらかく入金確認をしたい
- 「入金確認のお願いメール」を失礼なく送りたい

**フェーズ2: もう我慢の限界なので証拠つきで最終通告したい**
- 2〜3回催促しても反応がない
- 内容証明郵便や法的手続きを検討している
- 法的手前で揺れている

### カテゴリ説明

#### 1. やさしい催促（mild-reminder）
相手に角を立てずに支払いをお願いするための文面・手順を紹介。
- 入金確認メール
- 2回目催促メール
- 最終手前の催促

#### 2. 最終通告・法的手続き（final-warning）
内容証明郵便・最終通告・法的手前の動き方を解説。
- 内容証明の書き方
- 送るタイミング
- 法的手続きへの移行

#### 3. 全体フロー（about-process）
未入金対応の全体フロー、ステップ別の考え方、実務の注意点をまとめたガイド。

#### 4. 入金後の対応（thanks-and-closing）
入金ありがとうメール、入金受領連絡などの信用担保コミュニケーション（今後追加予定）。

## 記事の作成方法

### テンプレートを使う

`templates/guide.md`をコピーして、新しい記事を作成してください。

```bash
cp templates/guide.md content/guide/new-article.md
```

### Frontmatterの設定

```yaml
---
title: "記事タイトル（検索キーワードを含める）"
description: "記事の説明文（120〜160文字）"
date: "2025-10-31"
author: "編集部"
category: "mild-reminder"  # mild-reminder, final-warning, about-process
tags: ["入金確認", "催促メール"]
cta: "soft"  # soft, hard, mixed
---
```

### CTAの種類

- **soft**: 請求/入金管理SaaSの無料トライアル向け（mild-reminder記事用）
- **hard**: 債権回収・回収代行の無料相談向け（final-warning記事用）
- **mixed**: 両方のCTAを含む（about-process記事用）
