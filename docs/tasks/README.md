## 🚀 実装の進め方

### 基本原則

1. **順次実行**: Stage 1から順番に実施（並行実行不可）
2. **チェックリスト活用**: 各Stageのマークダウンファイル内のチェックボックスを埋める
3. **コミット**: 各Stageの完了時にコミット
4. **3回失敗ルール**: 同じタスクで3回失敗したら立ち止まり、アプローチを見直す

### 各Stageの進め方

#### 開始時

1. 該当Stageのマークダウンファイルを開く
2. 「Status」を `🔄 In Progress` に変更
3. 「Started」に現在の日付を記入
4. このREADME.mdの進捗表も更新

#### 作業中

1. タスクを1つずつ実行
2. 完了したら `- [ ]` を `- [x]` に変更
3. 定期的にコミット（少なくとも1日1回）

#### 完了時

1. すべての Success Criteria を満たしたか確認
2. すべての Tests が合格したか確認
3. 「Status」を `✅ Completed` に変更
4. 「Completed」に現在の日付を記入
5. このREADME.mdの進捗表を更新
6. 完了コミット作成

```bash
git add .
git commit -m "feat: complete Stage X - [名前]"
git push
```

## 🔍 トラブルシューティング

### よくある問題

#### Stageが完了したかわからない

→ 各Stageのマークダウンファイルの「Success Criteria」セクションをすべて満たしているか確認

#### タスクで3回失敗した

→ [IMPLEMENTATION_PLAN.md](../IMPLEMENTATION_PLAN.md) の「トラブルシューティング」セクションを参照

#### 依存関係がわからない

→ Stage 1 → 2 → 3 → 4 → 5 の順番で実施（前のStageが完了していないと次に進めない）
