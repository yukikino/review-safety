## ⚠️ 改善すべき点

### 1. 【重要度: 高】記事の公開日付が全て同一
**現状**: 全記事が `2025-11-03`
**問題**: 新鮮さシグナルが弱い、クローラーが全記事を同時に発見

**推奨対応**:
```markdown
# 記事公開日を分散させる
- playbook/complete-workflow.md: 2025-11-01（最優先記事）
- mild-response記事群: 2025-11-03, 11-05, 11-07（3日間隔）
- escalation記事群: 2025-11-10, 11-12
- industry-specific記事群: 2025-11-15, 11-17
```

**効果**:
- 定期的な更新をGoogleが認識
- 記事が徐々にインデックスされ、クロールバジェット効率化

### 2. 【重要度: 高】OG画像がプレースホルダー
**現状**: 全記事で `/og-default.png` (1318バイト = ほぼ空)
**問題**: SNSシェア時に視覚的インパクトなし、CTR低下

**推奨対応**:
1. **共通OG画像を作成** (1200x630px):
   - サイトロゴ
   - 「低評価口コミ・クレーム対応マニュアル」のテキスト
   - 「返信テンプレート」「削除依頼」などのキーワード
2. **記事ごとのOG画像生成** (将来):
   - タイトルを動的に画像化
   - Vercel OG Image Generationを活用

### 3. 【重要度: 中】業種別記事が不足
**現状**: 美容クリニック、飲食店のみ
**機会損失**: エステ、パーソナルジム、整体院など高需要業種が未カバー

**推奨対応**:
```
追加すべき記事:
- エステ・痩身サロン向け（薬機法注意）
- パーソナルジム向け（効果の断定NG）
- 整体院向け（医療類似行為の表現注意）
- マツエク・ネイルサロン向け
```

### 4. 【重要度: 中】ロングテールKWが未カバー
**現状**: 「待たされた」「態度が悪い」など主要KWは網羅
**機会**: より具体的な状況別記事でロングテール獲得

**推奨対応**:
```
追加記事案:
mild-response:
- 「料理がまずい」と書かれた時の対応
- 「予約が取れない」クレームへの返信
- 「駐車場がない」などの設備面クレーム

escalation:
- 「食中毒」と書かれた時の保健所対応含む返信
- 競合他社による嫌がらせレビューの見分け方
- 複数アカウントからの連続投稿への対応

industry-specific:
- 歯科クリニック向け（医療広告ガイドライン最厳格）
- ホテル・民泊向け（宿泊業特有のクレーム）
```

### 5. 【重要度: 中】内部リンクアンカーテキストの最適化
**現状**: 「続きを読む →」「こちら」など汎用的
**改善点**: キーワードを含むアンカーテキスト

**推奨対応**:
```tsx
// Before
<Link href="/escalation/false-claims">続きを読む →</Link>

// After
<Link href="/escalation/false-claims">
  事実と違う口コミへの対応方法を見る
</Link>
```

### 6. 【重要度: 低】FAQの拡充
**現状**: ホームページに8個のFAQ
**改善点**: 各記事にもFAQ Schemaを追加

**推奨対応**:
- 記事ごとに関連するFAQ 3〜5個を追加
- FAQ Schemaでリッチリザルト獲得を狙う

### 7. 【重要度: 低】パンくずリストの視覚表示
**現状**: Schema はあるが視覚的なパンくずなし
**改善点**: ユーザビリティとSEOの両面で効果

**推奨対応**:
```tsx
// components/breadcrumb.tsx で視覚的なパンくずを追加
<nav aria-label="breadcrumb">
  ホーム > mild-response > 記事タイトル
</nav>
```
